# Java按值引用
---
1. 如何理解什么是按值引用：
	1. 即有没有把参数复制一份传入方法中。
	2. Java都是按值引用
2. 传入的是指向对象的指针的复制，指向同一个对象，所以虽然是按值引用，但仍然可以在方法中影响方法外的值，主要的判断依据是有没有修改指针指向的堆上的对象。

```
void change(Integer num) {
    num = 999;  // 问题在这！
}

Integer count = 10;
change(count);
System.out.println(count);  // 输出 10，不是 999
```
在这个例子里，num = 999 其实会自动装箱 num = Integer.valueOf(999)，只是修改了栈帧里的局部变量赋值，没有改变堆中内容。


# null 的本质是什么
---
null 本质上是把引用变量的值中的该变量指向了 0 这个特殊地址
