- [[SADE]] 上课笔记：C4.4
	- interaction diagram
		- 干什么的：
			- 展现建模对象之间的交互行为（信息交换），特别是方法调用的时间顺序
		- 用哪些图来表示：
			- 序列图（Sequence diagram）
				- 图元素：（注意语法，即画中的标注）
					- X轴向右表示对象，Y轴向下表示时间
					- head of lifeline（表示role，role比object宽泛） P8
						- LATER role与object的区别是什么
					- body of lifeline
					- Execution Specification 虚线上的块，进入的箭头要指向块顶端
					  collapsed:: true
						- Message：两个块之间传递的信息 P9
						  collapsed:: true
							- 消息的不同类型的表示：
								- 同步消息：实线实三角，语法：msg(par1,par2)
								- 异步消息：实线实箭头
								- 响应消息：虚线实箭头，上面要标注 att=msg(par1,par2):val
								- object creation：虚线实箭头，指向矩形头部，要标注 <<create>>
								- object destruction：在body的虚线底部打叉，表示这个对象销毁
								- Found message：实线实箭头从实心小球指向某个类的body，表示发送者不知道
								- Lost message：实线实箭头从body指向实心小球，表示不知道接受者，不关心是谁接受了
					- Combined Fragment P13
					  collapsed:: true
						- 套在一组横向的箭头上的方块区域，表示这组交互的特殊操作，比如 loop
						- 有12种可用操作：
							- alt：就是switch操作，不同的情况画在左边，用 [] 标注，情况之间用虚线区别 P15
							- opt：if，如果[条件]为真，那么执行块中操作 P16
							- loop：注意条件和循环次数的写法 P17
								- **`min`（最小值）** ：循环体**至少执行的最小次数**（即使条件不满足，也会先执行 `min` 次）。
								- **`max`（最大值）** ：循环体**最多执行的最大次数**（即使条件仍满足，达到 `max` 次后也会终止）。
								- break：满足条件break
							- 并发：
								- par：这个块内并发执行，执行顺序不影响
									- critical：并发中的原子操作块，lua脚本，事务
								- strict：严格按照顺序执行
								- seq：弱顺序
									- **同一生命线（Lifeline）上的消息必须严格按顺序执行**（即同一对象的操作不能乱序）。
									- **不同生命线之间的消息可以重叠或重排序**，只要不违反因果依赖（如消息触发关系）。
							- 过滤与断言：
								- ignore：块内的执行顺序中，可以不考虑被ignore的消息 P29
								- consider：
								- assert：块内必须按照预期执行，否则视为系统错误或违反约束 P30
								- neg：块内的为不应该发生的操作，无效操作
					- Integrate：sd+ref，将一个sequence图打包，作为一个模块用到另一个图中 P32
					- Gate：作为interaction图的接口，可以接受边界外的信息，并输出到边界外
					- Continuation marker：将一组alt中的情况打包，用到另一个alt中 P36
